<task type="auto">
  <name>Task 1: application-benchmark.yml scheduler 활성화 + run-benchmark.sh --summary-export 추가</name>
  <files>src/main/resources/application-benchmark.yml, scripts/k6/run-benchmark.sh</files>
  <action>
  **application-benchmark.yml 수정:**
  파일 최하단(spring: 블록 밖)에 다음을 추가한다:

  ```yaml
  scheduler:
    enabled: true
  ```

  이유: SchedulerConfig는 @ConditionalOnProperty(name="scheduler.enabled", havingValue="true")로 선언되어 matchIfMissing이 없다. 이 키가 없으면 OutboxWorker의 @Scheduled가 실행되지 않아 PENDING 레코드가 영원히 처리되지 않는다. application.yml(기본 프로파일)에는 추가하지 않는다 — dev/test 환경에서 OutboxWorker 미실행은 의도적 설계다.

  **run-benchmark.sh 수정:**
  run_strategy() 함수 내부에 다음 두 변경을 적용한다:

  1. docker run 전에 results 디렉토리 생성:
  ```bash
  mkdir -p "${SCRIPT_DIR}/results"
  ```

  2. docker run 명령에 --summary-export 플래그 추가:
  ```bash
  docker run --rm \
    -v "${SCRIPT_DIR}:/scripts" \
    -e BASE_URL="${BASE_URL}" \
    grafana/k6 run \
      --summary-export="/scripts/results/${strategy}.json" \
      "/scripts/${script}"
  ```

  /scripts/results/ 경로는 기존 -v "${SCRIPT_DIR}:/scripts" 볼륨 내부이므로 추가 마운트 없이 호스트의 scripts/k6/results/ 디렉토리에 저장된다.

  **커밋:** 두 파일을 하나의 커밋으로 묶는다. 커밋 메시지(한글 본문):
  `fix(benchmark): application-benchmark.yml scheduler 활성화 + run-benchmark.sh JSON export 추가`
  </action>

    <automated>./gradlew test</automated>
  </verify>
  <done>
  - application-benchmark.yml에 scheduler.enabled: true가 최상위 레벨에 존재한다 (spring: 블록 밖)
  - run-benchmark.sh에 --summary-export="/scripts/results/${strategy}.json" 플래그가 docker run 명령에 포함된다
