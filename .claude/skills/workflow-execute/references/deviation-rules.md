# Deviation Rules 상세

## Rule 1 — 자동 수정 후 계속

아래 상황은 자동으로 수정하고 커밋 메시지에 내용을 추가한다.
- 컴파일 오류, 깨진 import
- 기존 테스트를 깨뜨리는 사이드 이펙트 (명백한 버그 수준)
- 오탈자, 잘못된 메서드명

```
git commit -m "feat: <태스크 이름> 구현

- <주요 변경>
- [Rule 1] FooException import 누락 수정"
```

---

## Rule 2 — 즉시 멈추고 확인 요청

아래 상황은 임의로 진행하지 않는다.
- 새 DB 테이블/컬럼 추가가 필요
- 레이어 경계를 넘는 의존성 추가 (예: domain이 infrastructure를 참조)
- build.gradle 의존성 추가
- 기존 포트 인터페이스의 시그니처 변경
- `@ConditionalOnProperty` 조건 변경

아래 형식으로 보고하고 승인을 기다린다.

```
## 아키텍처 변경 필요

**발견 위치**: Task N — <태스크 이름>
**상황**: <무엇을 하려다 발견했는지>
**필요한 변경**: <정확히 무엇을 바꿔야 하는지>
**영향 범위**: <파일/레이어>
**선택지**:
- A) <선택지와 트레이드오프>
- B) <선택지와 트레이드오프>

승인 후 진행하겠습니다.
```
