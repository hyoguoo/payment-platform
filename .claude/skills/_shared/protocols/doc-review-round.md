# 문서 검수 라운드 프로토콜

문서 콘텐츠(포스팅, 깃헙 위키, 리드미 등)의 품질을 4개 관점에서 병렬 검수한다.
최대 3회 루프를 돌며, 모든 관점이 PASS면 종료한다.

---

## 검수 관점 4가지

### 관점 1: 규격 준수

근거: `.claude/skills/_shared/references/writing.md`

문서가 프로젝트 작성 컨벤션을 지키는지 확인한다.

| 체크 항목 | 판정 기준 |
|:---:|:---:|
| 문체 `~다.` 또는 명사형 종결 | `~합니다/~요/~습니다` 없음 |
| 한 줄 한 문장, 길면 대시 리스트 전환 | 한 문장이 2줄 이상 이어지지 않음 |
| 표 셀 가운데 정렬(`:---:`) | 모든 표에 적용 |
| 표 셀 간격 임의 조절 금지 | 구분선 길이 변경 없음 |
| HTML 태그 사용 금지 | 순수 마크다운 문법만 사용 |
| 헤더 중심 구성 | 산문형 단락 대신 헤더 + 짧은 설명 조합 |

### 관점 2: 서사 일관성

문서의 논리 구조가 일관되는지 확인한다.

| 체크 항목 | 판정 기준 |
|:---:|:---:|
| 시간 순서 모순 없음 | 과거→현재 흐름이 역전되지 않음 |
| 용어 일관성 | 같은 개념에 다른 이름 사용 없음 |
| 서술 관점 통일 | 중간에 시점(1인칭/3인칭)이 바뀌지 않음 |
| 도입부-본문-결론 연결 | 도입에서 제시한 문제를 결론에서 회수 |
| 중복 서술 없음 | 같은 내용을 다른 섹션에서 반복하지 않음 |

### 관점 3: 기술 정확성

문서에 포함된 코드, 수치, 동작 방식이 실제와 일치하는지 확인한다.

| 체크 항목 | 판정 기준 |
|:---:|:---:|
| 코드 블록 정확성 | 실제 소스 코드와 대조해 불일치 없음 |
| 상태/타입/필드 목록 | 실제 enum, record, class와 대조 |
| 다이어그램 정합성 | Mermaid 전이/흐름이 코드 로직과 일치 |
| 수치/버전 정확성 | 구체적 수치나 버전이 검증됨 |

### 관점 4: 독자 친화성

프로젝트를 모르는 독자가 문서만으로 이해할 수 있는지 확인한다.

| 체크 항목 | 판정 기준 |
|:---:|:---:|
| 전문 용어 첫 등장 시 설명 | 약어, 프로젝트 고유 개념에 설명 있음 |
| 다이어그램 자기 설명성 | 참여자 이름, 화살표 레이블이 충분 |
| 코드 블록 전후 맥락 | "무엇을 왜 보여주는지" 설명 유무 |
| 배경 지식 전제 최소화 | 사전 지식 없이도 핵심 전제를 파악 가능 |
| 시각적 요소 적절 배치 | 텍스트만 이어지지 않고 표/다이어그램/코드로 환기 |

---

## 실행 흐름

```
루프 시작 (최대 3회)
  ├─ 관점 1: 규격 준수 → PASS / FAIL + 지적 사항
  ├─ 관점 2: 서사 일관성 → PASS / FAIL + 지적 사항
  ├─ 관점 3: 기술 정확성 → PASS / FAIL + 지적 사항
  ├─ 관점 4: 독자 친화성 → PASS / FAIL + 지적 사항
  ├─ 모두 PASS → 종료
  └─ FAIL 있음 → 지적 사항 반영 후 다음 루프
```

### 1회 루프 절차

1. 4개 관점을 순서대로 실행한다.
2. 각 관점의 체크 항목을 PASS/FAIL로 판정하고, FAIL 항목에 구체적 위치와 수정 방향을 기록한다.
3. 결과를 아래 출력 형식으로 정리한다.
4. FAIL 항목이 있으면 문서를 수정한 뒤 다음 루프를 진행한다.
5. 모든 관점이 PASS이거나 3회 루프를 소진하면 종료한다.

---

## 출력 형식

```
## 검수 라운드 N

### 관점 1: 규격 준수 — PASS / FAIL
| 체크 항목 | 판정 | 위치 | 수정 방향 |
|:---:|:---:|:---:|:---:|
| ... | PASS/FAIL | (해당 줄/섹션) | (FAIL 시 구체적 수정 방향) |

### 관점 2: 서사 일관성 — PASS / FAIL
(동일 형식)

### 관점 3: 기술 정확성 — PASS / FAIL
(동일 형식)

### 관점 4: 독자 친화성 — PASS / FAIL
(동일 형식)

### 종합
- PASS: N개 / FAIL: N개
- 다음 행동: (수정 후 재루프 / 종료)
```

## 적용 범위

이 프로토콜은 문서 유형에 관계없이 적용 가능하다.
문서 유형별로 관점의 가중치나 세부 체크 항목을 조정할 수 있다.

| 문서 유형 | 관점 1 | 관점 2 | 관점 3 | 관점 4 |
|:---:|:---:|:---:|:---:|:---:|
| 포스팅 | 필수 | 필수 | 필수 | 필수 |
| 깃헙 위키 | 필수 | 필수 | 필수 | 필수 |
| 리드미 | 필수 | 선택 | 필수 | 필수 |
