# docs/context/<TOPIC>.md 작성 형식

논의가 완료되면 아래 형식으로 문서를 작성한다.
모든 섹션이 필요하지는 않다 — 해당하는 것만 포함한다.

```markdown
# <주제> 설계

> 최종 수정: YYYY-MM-DD

---

## 문제 정의

현재 무엇이 문제인지, 왜 변경이 필요한지.

## 영향 범위

어떤 레이어/클래스가 영향을 받는지.
- 변경: ...
- 신규: ...
- 무관: ...

## 설계 옵션 비교

### Option A: <이름>
...
- **장점**: ...
- **단점**: ...

### Option B: <이름>
...
- **장점**: ...
- **단점**: ...

## 결정 사항

| 항목 | 결정 | 이유 |
|------|------|------|
| 구현 방식 | Option A | ... |
| 레이어 위치 | application | ... |

## 제외 범위

이번 작업에서 의도적으로 다루지 않는 것과 그 이유.
- <항목>: <이유>

## 참고

관련 다이어그램, 시퀀스, 외부 문서 링크.
```
