# 문서 검수 스킬

문서 콘텐츠를 4개 관점에서 검수한다. 각 관점은 독립 서브에이전트가 병렬로 수행하며, 최대 3회 루프를 돈다.
작성 컨벤션: `.claude/skills/_shared/conventions/writing.md`

## 검수 관점 4가지

### 관점 1: 규격 준수 (근거: `conventions/writing.md`)

| 체크 항목 | 판정 기준 |
|:---:|:---:|
| 문체 `~다.` 또는 명사형 종결 | `~합니다/~요/~습니다` 없음 |
| 한 줄 한 문장, 길면 대시 리스트 전환 | 한 문장이 2줄 이상 이어지지 않음 |
| 표 셀 가운데 정렬(`:---:`) | 모든 표에 적용 |
| 표 셀 간격 임의 조절 금지 | 구분선 길이 변경 없음 |
| HTML 태그 사용 금지 | 순수 마크다운 문법만 사용 |
| 헤더 중심 구성 | 산문형 단락 대신 헤더 + 짧은 설명 조합 |
| 메서드 호출 narrative 없음 | prose 본문에 `X.foo()` 형태 노출 없음 (T4 룰), 도메인 행위로 치환됨 |
| 동의어 통일 | `writing.md`의 동의어 통일 표 기준 일관 표기 |
| 약어 풀어쓰기 | 도메인 약어 첫 등장 시 1회 풀어쓰기 |

### 관점 2: 서사 일관성

| 체크 항목 | 판정 기준 |
|:---:|:---:|
| 시간 순서 모순 없음 | 과거→현재 흐름이 역전되지 않음 |
| 용어 일관성 | 같은 개념에 다른 이름 사용 없음 |
| 서술 관점 통일 | 중간에 시점(1인칭/3인칭)이 바뀌지 않음 |
| 도입부-본문-결론 연결 | 도입에서 제시한 문제를 결론에서 회수 |
| 중복 서술 없음 | 같은 내용을 다른 섹션에서 반복하지 않음 |

### 관점 3: 기술 정확성

| 체크 항목 | 판정 기준 |
|:---:|:---:|
| 코드 블록 정확성 | 실제 소스 코드와 대조해 불일치 없음 |
| 상태/타입/필드 목록 | 실제 enum, record, class와 대조 |
| 다이어그램 정합성 | Mermaid 전이/흐름이 코드 로직과 일치 |
| 수치/버전 정확성 | 구체적 수치나 버전이 검증됨 |

### 관점 4: 독자 친화성

| 체크 항목 | 판정 기준 |
|:---:|:---:|
| 전문 용어 첫 등장 시 설명 | 약어, 프로젝트 고유 개념에 설명 있음 |
| 다이어그램 자기 설명성 | 참여자 이름, 화살표 레이블이 충분 |
| 코드 블록 전후 맥락 | "무엇을 왜 보여주는지" 설명 유무 |
| 배경 지식 전제 최소화 | 사전 지식 없이도 핵심 전제를 파악 가능 |
| 시각적 요소 적절 배치 | 텍스트만 이어지지 않고 표/다이어그램/코드로 환기 |

## 실행 절차

### 1. 서브에이전트 4개 병렬 디스패치 (단일 메시지)

각 서브에이전트에 전달: 검수 대상 문서 경로 + 해당 관점의 체크 항목 표 + (규격 준수 관점에는 `conventions/writing.md` 경로, 기술 정확성 관점에는 "참조 소스 코드를 직접 읽어 대조" 지시).

프롬프트 템플릿:

```
문서 검수를 수행한다.
검수 대상: {문서 경로} / 검수 관점: {관점명}

아래 체크 항목을 각각 PASS/FAIL로 판정하고, FAIL 항목에는 구체적 위치(줄 번호 또는 섹션명)와 수정 방향을 기록한다.
체크 항목: {해당 관점의 체크 항목 표}

출력 형식:
### {관점명} — PASS / FAIL
| 체크 항목 | 판정 | 위치 | 수정 방향 |
|:---:|:---:|:---:|:---:|
```

### 2. 결과 수집 및 종합

```
## 검수 라운드 N
### 관점 1~4: <이름> — PASS / FAIL (각 결과)
### 종합
- PASS: N개 / FAIL: N개
- 다음 행동: (수정 후 재루프 / 종료)
```

### 3. 루프 판정

- 모든 관점 PASS → 검수 완료
- FAIL 존재 → 메인이 FAIL 항목 수정 후 다음 루프
- 3회 루프 소진 → 남은 FAIL 항목을 사용자에게 보고하고 종료

## 문서 유형별 관점 적용

| 문서 유형 | 규격 준수 | 서사 일관성 | 기술 정확성 | 독자 친화성 |
|:---:|:---:|:---:|:---:|:---:|
| 포스팅 | 필수 | 필수 | 필수 | 필수 |
| 깃헙 위키 | 필수 | 필수 | 필수 | 필수 |
| 리드미 | 필수 | 선택 | 필수 | 필수 |

서사 일관성이 선택인 유형에서는 해당 서브에이전트를 디스패치하지 않는다.
