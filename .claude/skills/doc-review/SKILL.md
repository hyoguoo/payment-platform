# 문서 검수 스킬

문서 콘텐츠를 4개 관점에서 검수한다.
각 관점은 독립 서브에이전트가 병렬로 수행하며, 최대 3회 루프를 돈다.

검수 프로토콜 상세: `.claude/skills/_shared/protocols/doc-review-round.md`
작성 컨벤션 상세: `.claude/skills/_shared/references/writing.md`

---

## 검수 관점과 서브에이전트

| 관점 | 서브에이전트 역할 | 검수 대상 |
|:---:|:---:|:---:|
| 규격 준수 | 컨벤션 검사기 | 문체, 표 정렬, 헤더 구조, HTML 금지 등 |
| 서사 일관성 | 논리 검토기 | 시간 순서, 용어 통일, 관점 통일, 중복 제거 |
| 기술 정확성 | 팩트 검증기 | 코드 대조, 다이어그램 정합, 수치 검증 |
| 독자 친화성 | 독자 시뮬레이터 | 용어 설명, 맥락 제공, 시각 요소 배치 |

---

## 실행 절차

### 1. 서브에이전트 4개 병렬 디스패치

4개 관점을 동시에 실행한다. 각 서브에이전트에 아래 정보를 전달한다.

- 검수 대상 문서 경로
- 해당 관점의 체크 항목 (`doc-review-round.md`에서 발췌)
- 규격 준수 관점에는 `writing.md` 경로 추가 전달

서브에이전트 프롬프트 템플릿:

```
문서 검수를 수행한다.

검수 대상: {문서 경로}
검수 관점: {관점명}

아래 체크 항목을 각각 PASS/FAIL로 판정하고, FAIL 항목에는 구체적 위치(줄 번호 또는 섹션명)와 수정 방향을 기록한다.

체크 항목:
{해당 관점의 체크 항목 목록}

{규격 준수 관점일 경우}
컨벤션 파일을 읽고 대조한다: .claude/skills/_shared/references/writing.md

{기술 정확성 관점일 경우}
문서에서 참조하는 소스 코드를 직접 읽어 대조한다.

출력 형식:
### {관점명} — PASS / FAIL
| 체크 항목 | 판정 | 위치 | 수정 방향 |
|:---:|:---:|:---:|:---:|
| ... | PASS/FAIL | (줄/섹션) | (수정 방향) |
```

### 2. 결과 수집 및 종합

4개 서브에이전트 결과를 모아 아래 형식으로 정리한다.

```
## 검수 라운드 N

### 관점 1: 규격 준수 — PASS / FAIL
(서브에이전트 결과)

### 관점 2: 서사 일관성 — PASS / FAIL
(서브에이전트 결과)

### 관점 3: 기술 정확성 — PASS / FAIL
(서브에이전트 결과)

### 관점 4: 독자 친화성 — PASS / FAIL
(서브에이전트 결과)

### 종합
- PASS: N개 / FAIL: N개
- 다음 행동: (수정 후 재루프 / 종료)
```

### 3. 루프 판정

- 모든 관점 PASS → 검수 완료
- FAIL 존재 → 메인 에이전트가 FAIL 항목을 수정한 뒤 다음 루프 진행
- 3회 루프 소진 → 남은 FAIL 항목을 사용자에게 보고하고 종료

## 문서 유형별 관점 적용

| 문서 유형 | 규격 준수 | 서사 일관성 | 기술 정확성 | 독자 친화성 |
|:---:|:---:|:---:|:---:|:---:|
| 포스팅 | 필수 | 필수 | 필수 | 필수 |
| 깃헙 위키 | 필수 | 필수 | 필수 | 필수 |
| 리드미 | 필수 | 선택 | 필수 | 필수 |

리드미처럼 서사 일관성이 선택인 경우, 해당 서브에이전트는 디스패치하지 않는다.
